# Code of Conduct

AuthModules contributors and maintainers are expected to keep project spaces professional,
inclusive, and focused on the work.

## Expected behavior

- Use welcoming and respectful language.
- Assume good intent while staying precise about technical concerns.
- Keep discussion focused on the issue, pull request, or project decision.
- Accept constructive feedback and provide it with concrete context.
- Respect project scope, repository boundaries, and maintainer decisions.

## Unacceptable behavior

- Harassment, intimidation, threats, or discriminatory language.
- Personal attacks, insults, or sustained disruption.
- Publishing private information without explicit permission.
- Sharing credentials, tokens, private infrastructure details, or private operational notes.
- Repeatedly pushing out-of-scope changes into focused issues or pull requests after a maintainer
  decision.

## Enforcement

Maintainers may remove comments, close issues, reject pull requests, restrict participation, or take
other reasonable moderation actions when behavior violates this Code of Conduct.

Enforcement decisions should be proportionate to the situation and focused on preserving a safe,
productive project environment.

## Reporting

Report Code of Conduct concerns through the relevant repository issue only when the report can be
made without exposing private or sensitive information. If a public issue would expose sensitive
details, request a private maintainer contact without including those details publicly.
